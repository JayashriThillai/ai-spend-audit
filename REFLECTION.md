# Project Reflection

## Overall Experience

This project was my first experience building a full-stack web application with deployment, database integration, and production workflows. Initially, the project felt overwhelming because many technologies were new to me, including Next.js, Supabase, deployment pipelines, and CI workflows.

Over time, the project became much more manageable as I started understanding how frontend, backend, databases, and deployment systems connect together.


## What Went Well

### Consistent Daily Progress

Working daily and maintaining GitHub commits helped prevent last-minute panic. The DEVLOG system also made it easier to track progress and decisions.


### Full-Stack Learning

The project provided hands-on experience with:
- React state management
- Next.js routing
- API routes
- Supabase integration
- deployment using Vercel
- GitHub Actions CI


### User Interviews

Conducting real user interviews significantly improved the product idea. The interviews revealed that many users do not consciously track AI subscription spending and often subscribe impulsively.


## Biggest Challenges

### Environment Variables

Understanding how `.env.local` works and how production deployment handles environment variables was initially confusing.


### Dynamic Routing

Implementing dynamic report pages using route parameters required multiple debugging attempts before working correctly.


### Database Permissions

Supabase Row Level Security initially blocked database inserts, which caused confusion while testing audit storage.


## Technical Decisions

### Why Next.js?

Next.js was selected because:
- simple routing
- easy deployment
- strong React ecosystem
- modern full-stack capabilities


### Why Supabase?

Supabase provided:
- simple database setup
- fast integration
- hosted backend services
- beginner-friendly workflow


### Why Lightweight AI Summaries?

A rule-based AI-style summary system was ultimately chosen because:
- faster development
- reliable output
- lower complexity
- reduced dependency on external inference APIs


## What I Would Improve With More Time

If more time was available, I would add:
- authentication
- historical analytics dashboards
- advanced AI recommendations
- automated subscription detection
- organization-level reporting
- better mobile responsiveness


## Key Takeaway

The biggest lesson from this project was learning how real software projects involve much more than writing frontend code. Deployment, debugging, documentation, user research, testing, and product thinking are all equally important parts of building usable software.
